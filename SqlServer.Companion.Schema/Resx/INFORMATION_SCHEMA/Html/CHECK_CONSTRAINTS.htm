CHECK_CONSTRAINTS

Contains one row for each CHECK constraint in the current database. This information schema view returns information about the objects to which the current user has permissions. The INFORMATION_SCHEMA.CHECK_CONSTRAINTS view is based on the sysobjects and syscomments system tables.

To retrieve information from these views, specify the fully qualified name of INFORMATION_SCHEMA view_name.

Column name Data type Description
CONSTRAINT_CATALOG nvarchar(128) Constraint qualifier.
CONSTRAINT_SCHEMA nvarchar(128) Constraint owner.
CONSTRAINT_NAME sysname Constraint name.
